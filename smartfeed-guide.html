02
What's Inside Each File
A closer look at what each file contains and what it's responsible for.
index.html The skeleton

The single webpage that IS the entire app. This defines what's on the page and in what order — but not how it looks or how it behaves.

What's inside: Hero section with address input & GPS button · Results section with zone banner, score cards, nutrition scanner, and nearby places grid · Profile section with calorie targets and favorites · How-it-works strip · Stats strip · Footer · Popup modals for login, signup, goals, and suggestions
style.css The skin

Every visual style for the entire app. Colors, fonts, spacing, animations, and mobile responsiveness. Zero logic — just how things look.

What's inside: Color variables (green = healthy, amber = swamp, red = desert) · Hero background gradient · Zone banner colors · Score card grid · Nutrition progress bars · Places card layout · Modal styling · Mobile breakpoint at 768px
app.js The brain

All the JavaScript logic. Every button click, API call, calculation, and screen change runs through here. This is where everything actually happens.

What's inside: Supabase & Google Maps setup · Login / signup / logout · Food desert detection using USDA + Census data · Zone classification (desert, swamp, or healthy) · Nearby places search with dietary & budget filters · Nutrition label scanner (photo upload + product search) · Calorie calculator using Mifflin-St Jeor equation · Favorites & community suggestions
api/config.js The key dispenser

A tiny backend function that securely hands the Google Maps and USDA API keys to the frontend. Keeps secrets out of your public code.

api/scan-label.js The AI tunnel

Receives a photo of a nutrition label from the user, sends it to Claude AI to read it, and returns the calories, protein, carbs, and fat back to the app. The Anthropic API key never leaves this server-side file.

vercel.json Deployment config

Tells Vercel two things: how to route API requests to the backend functions, and which security headers to set so the frontend can talk to the backend.

.env.example Secret key template

A reference showing which API keys the app needs (Google Maps, USDA, Anthropic). The real keys live in Vercel's environment variables — never in the code.

.gitignore Privacy guard

Tells Git to ignore secret files and build artifacts so they never get pushed to GitHub.

04
How It All Connects
The two main things a user can do — and how each one works behind the scenes.
Flow 1: Detect My Zone
1
User enters an address or clicks "Use My Location" → app.js converts it to coordinates using Google Maps
2
Coordinates are sent to the US Census Bureau to find the census tract
3
Census tract is checked against the USDA Food Access Research Atlas for official food desert data
4
Zone banner turns red (desert), amber (swamp), or green (healthy)
5
app.js searches Google Places for nearby grocery stores, farmers markets, and health food stores within half a mile
6
Results render as place cards — sorted by distance, filterable by diet and budget
Flow 2: Scan My Food
1
User uploads a photo of a nutrition label or searches a product by name
2
Photo path: app.js converts the image to base64 and sends it to api/scan-label.js
3
scan-label.js forwards the image to Claude AI, which reads the label and returns calories, protein, carbs, and fat as JSON
4
Search path: app.js queries the Open Food Facts API to find nutrition data by product name
5
app.js displays the results as color-coded progress bars showing what percentage of your daily target each macro represents

External Services
APIs and services SmartFeed connects to.
Supabase User accounts, database, favorites, and community suggestions
Google Maps Address lookup, nearby places search, and distance calculations
USDA Atlas Official government food desert and food access data
Census Bureau Converts GPS coordinates into census tract numbers (free, no key needed)
Claude AI Reads nutrition labels from uploaded photos
Open Food Facts Searches products by name for nutrition data (free, no key needed)
